# M1 Solution 骨架

本文件是 M1 阶段的"作战图"。它把 `Docs/Milestones.md` 中 M1 的建议任务拆成可勾选的最小项，并把 M1 中涉及的关键设计决策固化下来，避免后续阶段反复返工。

> 上下文恢复入口：阅读顺序为 `Docs/Architecture.md` → `Docs/Milestones.md` → 本文件。
>
> 本里程碑只搭骨架，不写业务逻辑；但凡是骨架就应当是正式的、可演进的版本，不允许"先这样以后再换"。

## 1. 目标

建立一套能够长期演进、面向 Bocchi 三段式架构的 .NET 解决方案骨架，使后续 M2 ~ M6 的实现可以直接挂载进来。

具体落点：

- 一个可构建、可测试、可运行的 `Bocchi.sln`。
- 一个能在内网启动并展示后台壳与健康检查的 Home Server。
- 一组与 `Docs/Architecture.md` 模块边界一一对应的核心库。
- 一组与核心库一一对应的测试项目。
- 工程层面的基础设施（仓库根级 MSBuild、包管理、代码风格、忽略规则、SDK 锁定）。
- 为后续 Theme 与 Cloud Server 预留目录与说明。

## 2. 范围与非目标

在范围内：

- 解决方案结构、目录布局、命名空间约定。
- Home Server 启动壳、健康检查端点、最小后台首页。
- 核心库的项目骨架与对外类型签名（接口/占位类型可以为空，但命名与归属必须正式）。
- 公共构建属性（TFM、Nullable、警告即错误、隐式 using、语言版本）。
- 集中包管理（Directory.Packages.props）。
- 单元测试与集成测试基础设施。
- 启动方式说明（README 中追加 Quickstart）。

非目标（移交给后续里程碑）：

- 任何真实的内容解析、扫描、存储逻辑（M2）。
- 任何真实的构建流水线、媒体处理、manifest（M3）。
- 任何完整的管理界面、内容列表、编辑器（M4）。
- 默认 Theme 的实际实现（M5）。
- RSS、Sitemap、搜索、发布（M6）。
- Cloud Server 的具体实现（M7）。

## 3. 关键决策

这些决策一旦在 M1 落定，后续不应"临时再换"。如确需调整，必须先更新本节再改代码。

| 主题 | 决策 | 依据 |
| --- | --- | --- |
| 目标框架 | `net10.0` 单一 TFM | 项目定位明确，只服务于 .NET 10 部署 |
| SDK 锁定 | `global.json` 锁定到当前可用的 .NET 10 SDK，`rollForward: latestFeature` | 保证 CI 与本机一致，又允许补丁升级 |
| Home Server UI 技术 | Blazor Web App，默认 InteractiveServer 渲染 | 后台为内网交互工作台，需要表单、实时日志、按 schema 生成配置 UI，Blazor Server 与 .NET 生态一致，避免另外维护 SPA 构建链 |
| 日志 | `Microsoft.Extensions.Logging` + Serilog（Console + File sink） | Serilog 是 .NET 事实标准，结构化日志在构建/发布日志阶段是刚需，越早接入越好 |
| 测试框架 | xUnit + `Microsoft.NET.Test.Sdk` + `FluentAssertions` | xUnit 是 .NET 现代标准，FluentAssertions 提高可读性；后续 Home Server 集成测试用 `Microsoft.AspNetCore.Mvc.Testing` |
| 包管理 | 集中包管理（`Directory.Packages.props`，`ManagePackageVersionsCentrally`） | 多项目共享版本，避免版本漂移 |
| 公共编译选项 | `Nullable=enable`、`ImplicitUsings=enable`、`TreatWarningsAsErrors=true`、`LangVersion=latest`、`AnalysisLevel=latest-recommended` | 一开始就把质量门槛拉满，避免后期降级 |
| 代码风格 | 仓库根 `.editorconfig`，沿用 .NET 默认规则集，4 空格缩进 | 统一团队/AI 协作时的格式 |
| 仓库忽略 | `dotnet new gitignore` 生成的 .NET 标准忽略 | 标准做法 |
| 程序集命名 | `Bocchi.<模块>`，例如 `Bocchi.HomeServer`、`Bocchi.ContentModel` | 与项目一致的命名空间根 |
| 解决方案布局 | `Src/HomeServer/`、`Src/Core/`、`Src/CloudServer/`、`Tests/`、`Themes/` | 与 `Docs/Architecture.md` §2 三段式直接对齐 |
| 核心库拆分 | `Bocchi.ContentModel`、`Bocchi.Workspace`、`Bocchi.GeneratorContract` | 内容事实、工作区抽象、Theme/Generator 契约三件套，对应 Architecture §3~§7 |
| Cloud Server 预留 | `Src/CloudServer/.gitkeep` + README 说明，不创建项目 | 与 Architecture §2.3、Milestones M7 保持一致：不提前实现 |
| Themes 目录 | 仓库根 `Themes/`，含 README 说明用途；M1 不放任何具体 Theme | 与 Architecture §7 对齐，避免在用户 workspace 与仓库内置 Theme 之间混淆 |

> 这些决策同时填补了 `Docs/Milestones.md` "待决问题" 中关于"Home Server 后台 UI 使用 Razor Pages、Blazor 还是混合方案"一项；M1 完成后会把对应条目从总文档"待决问题"中划掉。

## 4. 解决方案目标布局

```text
Bocchi/
  Bocchi.sln
  global.json
  Directory.Build.props
  Directory.Packages.props
  .editorconfig
  .gitignore
  README.md
  Docs/
  Src/
    HomeServer/
      Bocchi.HomeServer/
    Core/
      Bocchi.ContentModel/
      Bocchi.Workspace/
      Bocchi.GeneratorContract/
    CloudServer/
      README.md
  Tests/
    Bocchi.HomeServer.Tests/
    Bocchi.ContentModel.Tests/
    Bocchi.Workspace.Tests/
    Bocchi.GeneratorContract.Tests/
  Themes/
    README.md
```

依赖方向（严格单向，禁止反向引用）：

- `Bocchi.HomeServer` → `Bocchi.Workspace`、`Bocchi.GeneratorContract`、`Bocchi.ContentModel`
- `Bocchi.Workspace` → `Bocchi.ContentModel`
- `Bocchi.GeneratorContract` → `Bocchi.ContentModel`
- `Bocchi.ContentModel` → 仅依赖 BCL
- 每个测试项目仅引用对应的被测项目

## 6. 风险与回避

| 风险 | 回避 |
| --- | --- |
| 把 M2 ~ M6 的逻辑提前塞进 M1 | 任何核心库只保留类型签名，不写解析/扫描/序列化逻辑；超出 §5 的 PR 必须延后 |
| 集中包管理与 Razor SDK 隐式包冲突 | 对 Razor/ASP.NET Core 隐式包，需要在 Directory.Packages.props 中显式声明，或允许 `<EnableImplicitFrameworkReferences>` 处理；遇冲突时按官方文档加 `<PackageVersion>` 占位 |
| TreatWarningsAsErrors 导致模板默认警告报错 | 全量清理模板生成代码（Counter/Weather/示例）即可；保留的代码必须 0 警告 |
| Blazor InteractiveServer 在 .NET 10 模板下的项目结构变化 | 以 `dotnet new blazor` 实际产物为准，本文件第 4 节布局只约束包装目录，模板内部结构（Components/Pages 等）以模板默认为准 |

## 8. 验证记录

执行环境：Ubuntu (GitHub Actions runner)，.NET 10 SDK（`global.json` 锁定）。

- `dotnet restore Bocchi.sln` —— 通过。
- `dotnet build Bocchi.sln -c Debug` —— 通过，0 警告 0 错误（公共 props 已开启 `TreatWarningsAsErrors`）。
- `dotnet test Bocchi.sln -c Debug --no-build` —— 通过，覆盖 ContentModel/Workspace/GeneratorContract 占位测试与 HomeServer 集成测试（`/healthz`、`/`）。
- `dotnet run --project Src/HomeServer/Bocchi.HomeServer` —— 启动到 `http://127.0.0.1:5081`；`curl http://127.0.0.1:5081/healthz` 返回 `Healthy`；`curl http://127.0.0.1:5081/` 返回包含 "Bocchi Home Server" 的 HTML。

残留风险：

- Blazor InteractiveServer 在持续开发中可能需要补充 antiforgery / static asset 配置；M1 仅做骨架验证，集成测试覆盖 GET 路径。
- 集中包管理对 ASP.NET Core 隐式包做了显式 PackageVersion 声明；如未来升级 .NET 版本，需要同步刷新 `Directory.Packages.props`。
