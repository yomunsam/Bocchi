# M6 内容资产管理专项

本文记录文章编辑器插图、上传、粘贴、附件与内容级资产面板的专项规划。它是 M6 之后补上的内容创作体验专项，不改变 M6 localization 主线已完成的结论，也不属于 M7 Feeds / Search / Publish 主线。

## 1. 背景

当前 Bocchi 已经具备内容资产的底层事实：

- workspace 约定 Post / Work 使用目录型内容：`index.md` + `assets/`。
- Page 使用目录型内容：`pages/<slug>/index.md`，也可以自然携带 `assets/`。
- M6 多语言内容版本使用同一 localization group，同一目录下的 `index.md`、`index.en-US.md` 等语言 variant 应共享 `assets/`。
- Generator 已经能从 Markdown / frontmatter 收集媒体引用，把内容资产复制到 `output/public/media/...`，并在 theme input 中输出 media 信息；媒体派生输出仍是后续切片。
- `EditorDraftService` 已经有临时草稿目录和 `assets/`，并提供把上传或粘贴资产写入 draft assets 的基础能力。
- Markdown editor 的图片按钮、paste、drop 上传闭环按 CA-02 补齐；资产面板、附件上传、Figure 语义和派生媒体仍是后续切片。

因此本专项不应另起一套全站媒体库。第一阶段应补齐“当前内容组自己的 assets 管理”和“编辑器插入资产”的闭环。

## 2. 目标

1. 文章、页面、作品能在编辑器中插入图片；短文插图随 Note composer 单独设计。
2. 粘贴板图片、拖拽文件、文件选择上传都能写入当前内容组的 `assets/`，并把 Markdown 引用插入光标位置。
3. 编辑器提供当前内容组的资产管理面板，能查看、插入、复制路径、删除未引用资产。
4. 未保存的新内容先写入 editor draft assets；第一次保存到 workspace 时，assets 随内容一起迁移。
5. 已保存内容直接写入内容目录的 `assets/`。
6. 多语言 variant 共享同一内容组资产目录，不为每个语言文件拆一套资产。
7. 保持 workspace 是用户内容事实来源；资产元数据第一阶段不进入数据库。
8. 后续 Figure / lightbox / lazy load / 响应式图片能力基于语义 HTML 与媒体派生管线实现，不把编辑器改成富文本 block editor。

## 3. 非目标

- 不做全站媒体库或跨文章复用资产。
- 不做 CDN、对象存储、外部媒体托管。
- 第一阶段不做图片裁剪、压缩、缩略图、webp/avif 派生或响应式图片生成；这些能力归入后续媒体派生管线。
- 不做远程 URL 自动下载。
- 不做富文本式图片布局编辑器。
- 不引入正文 block AST 作为 Figure / lightbox / lazy load 的前置条件。
- 不做复杂附件权限系统。对外静态站点中的内容资产默认随站点发布。

## 4. 核心决策

### 4.1 资产归属到内容组

资产归属单位是内容组，而不是单个语言 variant。

```text
posts/2026/hello/
  index.md
  index.en-US.md
  assets/
    cover.jpg
    chart-en.png

pages/about/
  index.md
  index.en-US.md
  assets/
    avatar.jpg
```

理由：

- 同一内容的多个语言版本通常共享封面、截图和附件。
- 当前 M6 localization group 已经采用同目录 sibling 文件模型，`assets/` 自然属于这个目录。
- Generator 已经按内容目录解析相对媒体路径，继续使用 `assets/<file>` 最稳。

### 4.2 Note 使用目录型存储与短稳定 id

Note / 短文采用目录型存储。单文件 Note 是早期轻量文本模型，不再作为目标形态；后续实现不需要兼容旧单文件 Note。

workspace 目录负责按时间分桶，公开 URL 只暴露短稳定 id：

```text
notes/2026/0529/1432-k7p9xq2m/
  index.md
  assets/
    image.png

public URL:
/notes/k7p9xq2m/
```

目录规则：

```text
notes/{yyyy}/{MMdd}/{HHmm}-{id}/index.md
```

`id` 写入 frontmatter，作为 Note 的永久公开标识：

```yaml
---
id: k7p9xq2m
status: published
createdAt: 2026-05-29T14:32:00+08:00
updatedAt: 2026-05-29T14:32:00+08:00
---
```

规则：

- `id` 为 8 位小写字母与数字，创建时生成，永久不变。
- UI 不允许用户编辑 `id`。
- 目录名中的 `id` 必须与 frontmatter `id` 一致；不一致视为内容错误。
- 同一分钟创建多条 Note 时不需要时间后缀冲突处理，`id` 负责唯一性；如果目录已存在则重新生成 `id`。
- `yyyy/MMdd/HHmm` 只用于 workspace 浏览和 Git diff 组织，不进入公开 URL。
- SQLite 扫描投影如有需要，可以建立 `noteId -> sourcePath` 映射。
- Note 不参与 M6 内容源文件层多语言 variant 模型，不生成 `index.en-US.md`。未来翻译属于前台或 Cloud Server 展示/缓存能力。

Note media prefix 固定与公开 id 对齐：

```text
/media/notes/{id}/{file}
```

这样文件在 workspace 内移动时，只要 frontmatter `id` 不变，Note URL 和媒体 URL 都保持稳定。

短文目录化需要单独小切片处理：

- Workspace 初始化文档和 Architecture 更新。
- Note loader 只支持目录型 Note，并校验 `id` 格式、重复和目录一致性。
- Generator Note URL 输出为 `/notes/{id}/`。
- Generator media prefix 输出为 `/media/notes/{id}/...`。
- HomeServer Note 创建和编辑入口切换到目录型。

### 4.3 远程 URL 第一版只插入，不导入

远程 URL 可以作为 Markdown 链接或图片引用插入，但第一版不自动下载到本地 assets。

理由：

- 服务端下载远程 URL 会引入 SSRF、内网地址访问、重定向、超时、文件大小和 Content-Type 校验等问题。
- “插入远程 URL”与“导入远程文件”是两个不同产品动作，应分开实现。

后续如做导入，需要专门的安全边界：禁止私有网段、限制大小、限制重定向、限制 MIME、超时和错误展示。

### 4.4 资产面板不是媒体库

资产面板只管理当前内容组的 `assets/`。

它不负责跨内容搜索、复用、标签、图库或媒体中心能力。未来如果需要全站媒体库，应另立专项，不能把第一版资产面板扩成 DAM。

### 4.5 Markdown 图片与 Figure 语义分层

标准 Markdown 图片保持标准语义：`![alt](assets/file.jpg)` 中的 `alt` 是替代文本，不作为图注使用。上传、粘贴和拖拽插图的默认目标形态是空 alt：

```markdown
![](assets/file.jpg)
```

需要图注、尺寸倾向、对齐和点击放大时，使用 Bocchi Markdown Extension 的 Figure 语法：

```markdown
@[figure](assets/file.jpg){caption="Visual Studio" alt="新建 ASP.NET 项目的截图" size="small" align="center" zoom=true}
```

Figure 由统一 Markdown pipeline 渲染成语义 HTML，例如：

```html
<figure class="bocchi-figure bocchi-figure--small bocchi-figure--center" data-bocchi-lightbox>
  <a href="/media/posts/2026/demo/file.jpg">
    <img src="/media/posts/2026/demo/file.jpg" alt="新建 ASP.NET 项目的截图" loading="lazy" decoding="async">
  </a>
  <figcaption>Visual Studio</figcaption>
</figure>
```

规则：

- `caption` 控制可见图注；无 caption 时不输出 `figcaption`。
- `alt` 只用于无障碍替代文本；不会自动从 `caption` 复制。
- `size` / `align` 是展示倾向，由 Theme 转换为具体 CSS。
- `zoom` 是点击放大倾向，由 Theme JS / CSS 实现。
- Figure 语法必须被 Markdown 渲染、媒体引用收集、资产面板 referenced / unused 判断共同识别。

### 4.6 Theme 只消费语义 HTML，不解析正文 Markdown

现阶段 Theme input 继续同时提供 `markdown` 与 `html`，但公开 Theme 的正文渲染主路径是消费 `html`。Bocchi 自定义语法由 Workspace / Generator 的统一 Markdown pipeline 解析并渲染为稳定 HTML。

因此 Figure、lightbox、lazy load、占位和响应式图片不要求 Theme 解析 Markdown，也不要求引入 `body.blocks`。Theme 的适配职责是：

- 为 Bocchi 语义 class / data attribute 提供样式。
- 按 Theme config 控制 lightbox、lazy load、占位和 Figure 默认展示风格。
- 在正文 HTML 上做渐进增强，不改写内容事实。

只有当 Bocchi 进入真正 block editor、多端结构化正文或 Theme 需要重排正文 block 时，才重新评估 `body.blocks` / block AST。

### 4.7 媒体派生不进入 workspace

workspace 只保存原始媒体。jpg / png / gif 等源文件上传后保持原样进入当前内容组 `assets/`，webp、缩略图、预览图、响应式宽度档位等派生产物写入 `<data>/cache/derivatives/` 并在构建输出时进入 `/media/...`。

后续媒体派生管线负责：

- 为静态 jpg / png / webp / avif 生成 webp 响应式 variants。
- 为图片读取 width / height，用于减少前台布局跳动。
- 为预览图、小图和 lightbox 大图生成明确输出路径。
- 保留公开原始图的配置入口。
- 对 animated gif 优先生成 animated webp；无法可靠转换或转换收益差时保留原始 gif 作为输出 fallback。

媒体派生属于 Generator / 构建管线职责。Theme 只选择如何展示已生成的 variants，不直接处理图片转码。

## 5. UI 形态

### 5.1 Markdown toolbar

现有图片按钮保留，但行为升级：

- 点击图片按钮：打开文件选择，选择图片后上传并插入 `![](assets/file.ext)`。
- 后续附件阶段选择普通附件：插入 `[file.ext](assets/file.ext)`。
- 粘贴图片：上传到 assets，并插入图片 Markdown。
- 拖拽图片到编辑区：上传到 assets，并插入图片 Markdown。

按钮仍以图标为主，tooltip / aria-label 使用 Dashboard i18n。

### 5.2 编辑器资产面板

在 ContentEditor 右侧新增一个 Assets panel，与状态、语言版本 panel 同级。

分组：

| 分组 | 含义 |
| --- | --- |
| Referenced | Markdown / frontmatter 正在引用的本地 `assets/...` 文件。 |
| Unused | `assets/` 中存在但当前内容没有引用的文件。 |
| External | Markdown 中的远程 URL 图片或附件。第一版只展示，不管理。 |

每个本地资产项显示：

- 图片缩略图或文件类型 icon。
- 文件名。
- 文件大小。
- 引用状态。
- 操作：插入、复制路径、删除。

删除策略：

- 未引用资产可以直接删除或二次确认删除。
- 已引用资产不允许直接删除，必须先移除引用。

### 5.3 预览区图片 Inspector

预览区可以提供图片右键辅助编辑，但它不是富文本编辑器。右键菜单只修改 Markdown 源文本，随后由统一预览重新渲染。

第一版支持：

- 在预览区图片上打开自定义菜单。
- 从预览图片还原 `assets/...` 路径。
- 唯一匹配到 Markdown 图片或 Figure 时，允许编辑 alt / caption / size / align / zoom。
- 普通 Markdown 图片可以转换为 Figure。
- 同一资源在正文中出现多次、HTML `<img>` 或无法唯一定位时，提示回到 Markdown 源码编辑。

预览区菜单不直接修改 HTML，不保存独立 UI 状态，不引入 block AST。

## 6. 存储与服务边界

### 6.1 Draft 内容

新建内容在未落入 workspace 前使用：

```text
data/state/editor-drafts/<draft-id>/
  content.md
  session.json
  assets/
```

上传、粘贴、拖拽资产先写入 draft `assets/`。发布或保存为正式内容时，现有 `ContentEditingService.CreateFromDraftAsync` 会把 draft assets 移到最终内容目录。

### 6.2 已保存内容

已保存 Post / Page / Work 直接写入内容目录的 `assets/`。

如果保存时 slug 导致内容目录移动，现有目录移动逻辑会带走 `assets/`。

### 6.3 Note 内容

Note 使用目录型内容：

```text
notes/{yyyy}/{MMdd}/{HHmm}-{id}/
  index.md
  assets/
```

`id` 是 Note 的公开稳定标识，必须写入 `index.md` frontmatter。Note 的公开 URL 和后续动态服务器逻辑都使用 `id`，不使用 workspace 日期目录。不要为单文件 Note 开旁路资产目录。

## 7. 工作包

### CA-01 Asset Service

目标：建立 HomeServer 内容资产服务，封装文件系统操作。

职责：

- 解析当前编辑对象的资产目录：draft 或 workspace 内容目录。
- 枚举 `assets/` 文件。
- 写入上传文件并生成安全唯一文件名。
- 删除资产。
- 根据当前 Markdown / frontmatter 区分 referenced / unused。
- 返回 UI 需要的 asset view model。

验收：

- 路径穿越被拒绝。
- 文件名被规范化。
- 同名文件自动加后缀。
- draft assets 和 workspace assets 都可用。
- 没有数据库依赖。

### CA-02 Markdown Editor Upload Bridge

目标：让 CodeMirror paste/drop/file picker 能把浏览器 File 交给 Blazor，再插入返回的 Markdown。

职责：

- JS 捕获 paste/drop 的 File。
- 调用 .NET 上传方法，而不是只插入占位路径。
- 上传成功后在光标处插入 `![](assets/file)`；附件阶段插入 `[file](assets/file)`。
- 失败时给出 Dashboard i18n 错误提示。

验收：

- 粘贴图片能在 draft 中生成真实文件。
- 拖拽图片能生成真实文件。
- 文件选择上传能生成真实文件。
- 预览 HTML 能立即显示本地图片。

实施记录（2026-06-11）：

- CA-02A 已完成：Markdown editor paste image 会通过 Blazor stream bridge 写入当前内容组 `assets/`，并插入 `![filename](assets/filename.ext)`。
- CA-02B 已完成：同一条 image upload bridge 覆盖 drop 与 toolbar file picker；仍只开放 Post / Page / Work，不开放 Note composer。
- 上传入口继续走 `ContentAssetService`，不绕过服务层白名单、安全命名、同名去重和 draft / saved content assets 边界。
- 后续 Figure 语义落地时，上传插入文本收敛为空 alt 的 `![](assets/file)`，避免把文件名误写成替代文本。

### CA-03 ContentEditor Assets Panel

目标：在编辑器右侧加入当前内容组资产面板。

职责：

- 显示 referenced / unused / external。
- 支持插入现有资产。
- 支持复制相对路径。
- 支持删除未引用资产。
- 对已引用删除给出明确保护。

验收：

- 单语言、无资产内容不增加明显负担。
- 有资产内容能清楚看到当前文件是否被正文引用。
- UI 文案全部走 Dashboard i18n。

### CA-04 Note Directory Model

目标：把短文固定为目录型内容，并用 8 位短 id 作为公开 URL 与后续动态能力的稳定锚点。

职责：

- 定义 `notes/{yyyy}/{MMdd}/{HHmm}-{id}/index.md` workspace 结构。
- 创建时生成 8 位小写字母数字 `id`，写入 frontmatter，并用于目录名。
- Note loader 只支持目录型 Note。
- 校验 Note `id` 缺失、格式非法、重复、目录 id 与 frontmatter id 不一致。
- Note 创建入口写入目录型 Note。
- Generator Note URL 输出为 `/notes/{id}/`。
- Generator media path 输出为 `/media/notes/{id}/...`。
- 文档更新 Architecture / Milestones。

验收：

- 新 Note 可以插入图片。
- Note URL 不暴露日期目录。
- Generator 输出 note media。
- 现有 note tests 覆盖目录型。

实施记录（2026-05-29）：

- 已进入 CA-04 实施：Workspace scanner / loader 改为只接受 `notes/{yyyy}/{MMdd}/{HHmm}-{id}/index.md`。
- Note 创建入口生成 8 位小写字母数字 `id`，并写入目录名与 frontmatter。
- Generator / default-static 使用 `/notes/{id}/` 作为 Note 详情 URL，Note 媒体输出到 `/media/notes/{id}/...`。
- 不保留旧单文件 Note fallback；发现 `notes/{year}/*.md` 会作为校验错误报告。

### CA-05 Attachment Types

目标：在图片之外支持普通附件。

第一版允许：

- 图片：jpg、jpeg、png、gif、webp、avif。
- 文档附件：pdf、txt、md。
- 压缩包不进入第一版附件白名单。

SVG 第一版禁止通过编辑器上传。虽然项目已有静态输出 MIME 支持，但 SVG 作为用户上传内容存在脚本和外链风险；后续可以通过“受信任用户上传”或 sanitizer 决策放开。

验收：

- 图片插入为 image markdown。
- 非图片附件插入为 link markdown。
- MIME / extension 不符合预期时拒绝。

### CA-06 Bocchi Figure Syntax

目标：增加面向内容图片的 Bocchi Markdown Extension，支持 Figure、图注、展示倾向和点击放大倾向。

职责：

- 支持 `@[figure](assets/file.jpg){caption="..." alt="..." size="small" align="center" zoom=true}`。
- Markdown pipeline 将 Figure 渲染为稳定 `<figure>` 语义 HTML。
- 媒体引用收集识别 Figure 中的本地 `assets/...`。
- `ContentAssetService` referenced / unused 判断识别 Figure。
- default-static 为 `.bocchi-figure`、size / align class 和 `data-bocchi-lightbox` 提供基础样式与增强。

验收：

- Figure 中的本地图片会被复制到 `/media/...`。
- Figure 图注显示在图片下方，未设置图注时不输出空图注。
- Theme 不需要解析 Markdown 即可展示 Figure。
- 普通 Markdown 图片仍按标准图片渲染。

### CA-07 Preview Image Inspector

目标：在 Dashboard 编辑器预览区提供图片右键辅助编辑。

职责：

- JS 捕获预览区图片右键，打开 Dashboard 风格自定义菜单。
- 根据预览图片还原 `assets/...` 引用路径。
- 唯一匹配 Markdown 源文本时，支持编辑 alt / caption / size / align / zoom。
- 支持将普通 Markdown 图片转换为 Figure。
- 无法唯一定位时给出 i18n 提示，不猜测修改。

验收：

- 右键菜单只修改 Markdown 源文本，不直接修改预览 HTML。
- 修改后 CodeMirror 与预览同步更新。
- 同一资源多次引用时不会误改。
- UI 文案全部走 Dashboard i18n。

### CA-08 Media Derivatives

目标：构建时为内容媒体生成可发布派生产物，支持 webp、缩略图、响应式图片、占位和 lightbox 大图。

职责：

- 源文件继续保存在 workspace `assets/`，不写回派生产物。
- 派生产物写入 `<data>/cache/derivatives/`，并作为构建 artifact 输出到 `/media/...`。
- 为图片记录 width / height、原始路径、展示路径、lightbox 路径和 responsive variants。
- 静态 jpg / png / webp / avif 生成 webp variants；animated gif 优先生成 animated webp，失败或收益差时保留 gif fallback。
- Markdown / Figure 渲染输出 `loading="lazy"`、`decoding="async"`、必要的 width / height、`picture` / `srcset` 或等价语义 HTML。
- Theme config 控制 lightbox、占位、lazy load 风格和默认 Figure 展示倾向；Generator config 控制派生图开关、质量、宽度档位和原始图公开策略。

验收：

- workspace 扫描继续把缩略图、预览图和系统派生命名 variants 视为不应入库的 warning。
- 构建输出包含派生媒体 artifact，`.bocchi-manifest.json` 可追踪。
- 前台图片首屏布局不因图片加载明显跳动。
